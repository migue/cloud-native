Evaluation

The application will be deployed using the infrastructure provided by AWS, trying to adhere to the set of good practices we've covered during our lessons and videos.

A few things that need to be considerered:

      Your application should scale horizontally. We should easily add or remove new instances to our cluster depending on the needs
      Fault tolerant: our application should be available even if one of the machines is down. We need to do our best to keep the system running
      Persistency: the data stored by our application needs to be safely persisted and we cannot afford to lose data if, for example, our application crash

What needs to be delivered?

      A document (no predefined format) explaining the main decisions you've made to deploy your application and the selected architecture. You should talk about things like:
          How your application scales
          How fault-tolerance is achieved
          What happens in the presence of failures
          How a new version of your software is deployed
      A Terraform script(s) which allow me to reproduce your existing setup
